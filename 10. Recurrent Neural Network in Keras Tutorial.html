In this tutorial, we will test various Recurrent Neural Networks to predict monthly sunspots.¶

Data can be found using the following link:: https://machinelearningmastery.com/time-series-datasets-for-machine-learning/

Recurrent neural networks are used for sequential data like time series, speech, text, and financial. RNN's have become increasingly popular in recent years and have enormous predicting power. An RNN differs from a feed-forward neural network because information cycles through a loop. In other words, an RNN remembers information from the past.

Below is an example of the difference between a feed forward neural network and an RNN.

Defining functions¶

Create a function to build a recurrent neural network. Specifying the model as Sequential allows us to create our model layer by layer. We add a simple Recurrent Neural Network with specified parameters for the number of hidden units, dense units, input shape, and activation function. Then we add a dense layer, which means each neuron in the dense layer receives input from all neurons of its previous layer. The loss function we are looking to minimize is mean sqaured error.

In [31]:
def create_RNN(hidden_units, dense_units, input_shape, activation):
    model = Sequential()
    model.add(SimpleRNN(hidden_units, input_shape=input_shape, 
                        activation=activation[0]))
    model.add(Dense(units=dense_units, activation=activation[1]))
    model.compile(loss='mean_squared_error', optimizer='adam')
    return model

The function below reads in data from a csv file which can be found using the link: https://raw.githubusercontent.com/jbrownlee/Datasets/master/monthly-sunspots.csv. The data is scaled using min-max transformation. Min-Max scaling transforms every value between 0 and 1 by using the formula (x - min(x)) / (max(x) - min(x)). Then the data is split into training and test. The default split percent is 0.7, which means that 70% of data will be used for training and 30% for test. The split percent can be changed depending on the data or for testing purposes.

In [32]:
def get_train_test(csv, split_percent=0.7):
    df = pd.read_csv(csv, usecols=[1], engine='python')
    df = df.tail(int(len(df) * .25))
    data = np.array(df.values.astype('float32'))
    # Scale data using min-max transformation
    scaler = MinMaxScaler(feature_range=(0, 1))
    data = scaler.fit_transform(data).flatten()
    n = len(data)
    # Point for splitting data into train and test
    split = int(n*split_percent)
    train_data = data[range(split)]
    test_data = data[split:]
    return train_data, test_data, data

def get_XY(dat, time_steps):
    # Indices of target array
    Y_ind = np.arange(time_steps, len(dat), time_steps)
    Y = dat[Y_ind]
    # Prepare X
    rows_x = len(Y)
    X = dat[range(time_steps*rows_x)]
    X = np.reshape(X, (rows_x, time_steps, 1))    
    return X, Y

Creating a function to return both train and test rmse. The best performing model will have the lowest test rmse.

In [34]:
test_rmse_values = []
def print_error(trainY, testY, train_predict, test_predict):    
    # Error of predictions
    train_rmse = math.sqrt(mean_squared_error(trainY, train_predict))
    test_rmse = math.sqrt(mean_squared_error(testY, test_predict))
    # Print RMSE
    print('Train RMSE: %.3f RMSE' % (train_rmse))
    print('Test RMSE: %.3f RMSE' % (test_rmse)) 
    test_rmse_values.append(test_rmse)
    return test_rmse

def plot_result(trainY, testY, train_predict, test_predict):
    actual = np.append(trainY, testY)
    predictions = np.append(train_predict, test_predict)
    rows = len(actual)
    plt.figure(figsize=(15, 6), dpi=80)
    plt.plot(range(rows), actual)
    plt.plot(range(rows), predictions)
    plt.axvline(x=len(trainY), color='r')
    plt.legend(['Actual', 'Predictions'])
    plt.xlabel('Observation number after given time steps')
    plt.ylabel('MA scaled')
    plt.title('Actual and Predicted Values. The Red Line Separates The Training And Test Examples')

Combine functions to create one function that will create a recurrent neural network, split data, fit, predict, analyze and plot. Different activation functions can be specified for the first and second hidden layers, both with a default activation function of linear. Next, the number of time steps to be used in the model can be changed. Epochs can also be changed from a default value of 10. An epoch means training the neural network with all the training data for one cycle. In an epoch, we use all of the data exactly once.

In [11]:
def run_rnn_model(activation = "linear", activation2 = "linear", num_timesteps = 1, optimizer = "adam",epochs = 10):
    #demo_model = create_RNN(2, 1, (3,1), activation=['linear', 'linear'])
    csv = "https://raw.githubusercontent.com/jbrownlee/Datasets/master/monthly-sunspots.csv"
    train_data, test_data, data = get_train_test(csv)
    time_steps = num_timesteps
    trainX, trainY = get_XY(train_data, time_steps)
    testX, testY = get_XY(test_data, time_steps)
    model = create_RNN(hidden_units=3, dense_units=1, input_shape=(time_steps,1), 
                   activation=[activation, activation2])
    model.fit(trainX, trainY, epochs=epochs, batch_size=1, verbose=2)
    # make predictions
    train_predict = model.predict(trainX)
    test_predict = model.predict(testX)
    # Mean square error
    print_error(trainY, testY, train_predict, test_predict)
    print("first hidden layer activation = " + activation + ", second layer activation = " + activation2 + ", time steps = " + str(time_steps))
    plot_result(trainY, testY, train_predict, test_predict)

Testing different activation functions on each hidden layer as well as different time step values using nested for loops. The activation functions tested are linear, relu, and tanh. The time step values tested are 1, 2, and 3. We also test epoch values of 15 and 20. Each time a model is created lists get populated to store what the parameter values as well as the associated test rmse.

Summary¶

In this tutorial, we see the predictive power of RNN's on full display. The final model performed well in detecting seasonality and accurately predicting future monthly sunspots. Writing for loops to test different parameters is similar to using GridSearch. Given the testing, the final model could be used with confidence that it's both reliable and accurate.

To improve the model, we could test different optimizers. Changing optimizers could allow the model to fit the small discrepencies in the data better. We could also try adding an LSTM layer to the neural network.
